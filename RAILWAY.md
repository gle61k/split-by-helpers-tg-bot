# Деплой бота на Railway (пошагово)

Сделайте по порядку. После деплоя бот работает **без вашего компьютера** — можно пользоваться только с телефона.

---

## Часть 1. GitHub

### 1.1. Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) → **New repository**.
2. Имя, например: `telegram-hours-bot` (любое).
3. Репозиторий **Public** или **Private** — как хотите.
4. **Не** ставьте галочки «Add README / .gitignore» (у нас уже есть файлы).
5. Нажмите **Create repository**.

### 1.2. Залейте код с компьютера

Откройте **cmd** или **PowerShell** в папке проекта (где лежат `tg_bot.py`, `README.md`).

Проверьте, что **нет** файла `.env` в коммите (он в `.gitignore` — так и должно быть).

Команды (подставьте **ваш** URL репозитория с GitHub):

```bash
git init
git add .
git status
```

Убедитесь, что в списке **нет** `.env`. Если он есть — не коммитьте:

```bash
git reset HEAD .env
```

Дальше:

```bash
git commit -m "Telegram bot: split_by_helpers + Railway"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ИМЯ_РЕПО.git
git push -u origin main
```

GitHub попросит логин/пароль или **Personal Access Token** (для HTTPS токен вместо пароля).

## Часть 2. Railway

### 2.1. Аккаунт и проект

1. Зайдите на [railway.app](https://railway.app), войдите (можно через GitHub).
2. **New Project** → **Deploy from GitHub repo**.
3. Разрешите Railway доступ к репозиториям (если спросит).
4. Выберите репозиторий с этим ботом.
5. Railway начнёт сборку (Nixpacks подхватит Python и `requirements.txt`).

### 2.2. Переменные окружения (обязательно)

1. Откройте ваш **сервис** (клик по деплою) → вкладка **Variables**.
2. Добавьте переменную:

| Имя | Значение |
|-----|----------|
| `TELEGRAM_BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather) (строка вида `123456:ABC-DEF...`) |

3. Сохраните. Railway **пересоберёт/перезапустит** сервис — это нормально.

**Больше ничего вручную для URL задавать не нужно:** у Railway есть `RAILWAY_PUBLIC_DOMAIN` и `RAILWAY_PROJECT_ID`, бот сам включит режим webhook.

### 2.3. Публичный адрес (домен)

1. В том же сервисе откройте **Settings** → блок **Networking** / **Public Networking**.
2. Включите доступ: **Generate Domain** (или привяжите свой домен).
3. Появится URL вида `https://что-то.up.railway.app` — **запомните**, он должен быть с **https**.

Без публичного домена Telegram **не сможет** слать webhook, и бот не получит сообщения.

### 2.4. Проверка, что сервис живой

В браузере откройте:

`https://ВАШ_ДОМЕН.up.railway.app/`

Должно отобразиться слово **`ok`**. Если ошибка или «Application failed» — смотрите **Deployments → View logs**.

### 2.5. Логи

Вкладка **Deployments** → последний деплой → **View logs**.

Ищите строки:

- `Вебхук установлен: https://.../webhook`
- `Подключено к Telegram: @...`

Если есть ошибка про `PUBLIC_URL` или «нет публичного URL» — вернитесь к шагу **2.3** и сгенерируйте домен.

---

## Часть 3. Telegram

1. Откройте **именно того** бота, для которого токен в Railway (имя совпадает с тем, что в логах `@...`).
2. Отправьте файл выгрузки: **скрепка → Файл** → `.xls` или `.xlsx`.
3. Должен прийти ответ и готовые файлы.

Команда `/start` не обязательна.

---

## Важно: не запускайте два режима с одним токеном

- Если бот **на Railway** (webhook), **не** запускайте одновременно `run_bot.bat` / `python tg_bot.py` на ПК с **тем же** токеном — будет конфликт (polling сбивает webhook или наоборот).
- Для работы **только с телефона** держите **только Railway**, локальный скрипт не запускайте.

## Частые проблемы

| Симптом | Что сделать |
|---------|-------------|
| Бот молчит | Проверьте логи Railway, домен, переменную `TELEGRAM_BOT_TOKEN`, что пишете **тому** боту, чей токен в Variables. |
| `ok` в браузере не открывается | Сервис упал — смотрите логи сборки и рантайма. |
| Ошибка про порт | Railway сам задаёт `PORT`; трогать не нужно. |
| Мало файлов на выходе | В выгрузке должна быть колонка группы; иначе всё уходит в один `УР_Остальные` (логика `split_by_helpers.py`). |

## Версия Python

В корне репозитория файл `runtime.txt` (сейчас `python-3.11.9`). При ошибке сборки в Railway → **Variables** можно добавить `NIXPACKS_PYTHON_VERSION` = `3.11`.

---

Готово. Если что-то падает — скопируйте **текст ошибки из логов Railway** (без токена) и разберите по строкам сверху.
